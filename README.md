# 🎯 微软客户POE文档生成器 - 用户使用指南

欢迎使用微软客户 POE 文档生成器！这是一个基于 Azure OpenAI 驱动的一站式售前解决方案内容自动化生产工具。本指南将帮助你快速上手。

## ⚙️ 准备工作
确保工程目录 (`templates/`) 下包含必要的模板文件：
- `solution_template.docx.docx` (解决方案架构文档模板)
- `pov_template.docx` (POV部署计划模板)
- `AzureMigrateimporttemplate.csv` (Azure Migrate CSV 模板)

然后在项目根目录输入启动命令：
```bash
python3 -m streamlit run app.py
```

## 🚥 四步标准工作流

### 第 1 步：全局信息录入
无论生成什么文档，都要先提供客户的全局信息：
- **客户尊称**：例如“宇宙无敌科技有限公司”。
- **预估年消耗 (USD)**：填入客户潜在或预期能消耗的 Azure 估值规模，例如 `$50k+`，系统会根据这个预算智能决策大模型架构（大预算出 o1，低预算出 4o-mini）。
- **前线战报 (客户背景信息)**：把打听到的关键业务痛点、现有架构痛点丢进这里，AI 自动帮你写进背景需求里。

### 第 2 步：生成【解决方案文档】 (Tab 1)
这是整个流程的**核心和基础**。
1. 切换到「AI 解决方案文档」标签页。
2. 点击 **🚀 召唤神龙 (生成解决方案文档)** 按钮。
3. 等待生成完毕后，你会在右侧看到文档的实时 Markdown 预览。
4. 满意后，点击 **💾 疯狂输出 (下载解决方案 .docx)** 按钮，直接下载排版干净的 Word 文档。

> **⚠️ 强提醒**：此文档是后续所有内容的基础（也就是 Context），**你必须先生成它，才能使用后续的其他 Tab 功能！**

### 第 3 步：生成【POV 部署计划】 (Tab 2)
1. 切换到「POV 部署计划」。
2. 日历模块内微调选择一下**测试开始与结束的日期**。
3. 修改**乙方（我们）**负责执行测试项目的相关人员名字、头衔（如果你需要）。客户侧的对接人 AI 会根据第一步输入的背景自动编排。
4. 点击 **🗓️ 安排得明明白白** 生成，并在右侧预览、验证内容无误后下载 Word。

### 第 4 步：生成【SVG 架构图】 (Tab 3)
为了应对快速做汇报的场景：
1. 切换到「架构图 (SVG)」。
2. 点击 **🖌️ 灵魂画手附体** ，AI 将自动将步骤 2 生成的解决方案翻译成一张白底、美观且严格对正交走线进行网格布局的方案架构矢量图。
3. 右侧展示预览，底部提供 SVG 格式下载按钮。
4. 你也可以展开“查看 SVG 代码”，点击蓝色的“复制 SVG 代码”按钮，将代码直接通过任意画图工具（例如 Draw.io 的 `Edit -> Edit Geometry/SVG` 或者 Excalidraw）渲染贴进 PPT 里。

### （可选）特权操作：Azure Migrate CSV 生成 (Tab 4)
当遇到本地有老算力，想一并迁移上云的盘子时使用：
1. 请提供含有估算后 Azure 服务器实例数/价格的 `Excel 报价单`。
2. 上传表格，点击生成。
3. AI 负责把你杂乱的表格资源进行还原提取，自动推演倒推“客户本地环境”应该有几颗 CPU，几 GB 内存，甚至配几块存储盘多少的 IOPS！
4. 帮你拼接、填满那个晦涩难懂的 `AzureMigrateimporttemplate.csv` 下载上机导入。

## 🧹 工具重置
任何时候当你接了一个新客户想要一干二净重新开始，点击屏幕左侧栏底部的 **🧹 一键毁灭证据** 操作，所有内存数据瞬间洗白白。
